# v6.0 记忆系统 — 长期常驻学习 设计文档

## 目标

将 NotMe 从"单次 demo"升级为**持续成长的长期常驻学习 Agent**。完善记忆系统，实现零预训练纯净模式，Agent 从与用户的对话中在线学习一切语言和知识。

## 核心原则（不违背）

1. 涌现优于工程 — 记忆分化从 Hebb 动力学中自然涌现
2. 统计优于符号 — 不预设知识标签，一切从共现统计中学习
3. 身体是基础 — 情感门控决定记忆编码强度
4. 记忆是核心 — 一切认知能力都是记忆检索+模式补全
5. 自由能是唯一目标 — 记忆压缩最小化自由能
6. 脑区是蓝图 — 按人脑记忆系统分类设计架构
7. 并行优于串行 — 分布式表征，冗余存储
8. 预测优于反应 — 记忆 = 生成模型的参数
9. 瓶颈是特征不是 bug — 工作记忆的容量限制是结构性的

## 第一节：系统巩固 — 跨会话记忆转移

### 两个时间尺度

**会话内巩固**（增强已有机制）:
- 每 100 步：海马回放 + 模式分离 + 弱簇修剪
- 每轮对话后：情绪门控重放（高唤醒轮次额外重放）
- 每 10 轮：微量巩固（已有）

**跨会话系统巩固**（全新）:
```
会话结束时:
  海马情景记忆 (512维全量) → 皮层语义记忆 (64维主旨)
  
  算法:
  1. 选择海马中激活度最高的 N 个情景簇
  2. 提取"主旨向量" — 只保留 text[0:64] + body快照 + 情感
     丢弃视觉/听觉/痛觉细节
  3. 在语义网络中 recall() 匹配
  4. 已有知识 → 低学习率(0.01)更新; 全新 → 创建新簇
```

### 记忆痕迹时间梯度

| 时间 | 海马依赖 | 皮层依赖 | 说明 |
|------|---------|---------|------|
| 学习后 1 小时内 | 100% | 0% | 会话内，完全海马依赖 |
| 1 天后 | 60% | 40% | 第一次跨会话巩固 |
| 1 周后 | 20% | 80% | 多次巩固后皮层主导 |
| 1 月后 | 5% | 95% | 语义化，几乎不依赖海马 |

对应标准巩固模型（Squire & Alvarez, 1995）和多重痕迹理论（Nadel & Moscovitch, 1997）。

## 第二节：程序性记忆 — 纹状体习惯学习

### DMS ↔ DLS 转移

```
DMS (背内侧纹状体) — 目标导向         DLS (背外侧纹状体) — 习惯
尾状核 + 前额叶                      壳核 + 感觉运动皮层
A→O 学习: "我知道这样做会得到什么"     S→R 学习: "我不假思索就这么做了"
灵活、可迁移                          僵化、自动化
────────────────────────────────────────────────────────
              重复 → 从左向右转移
```

### Striatum 实现

**存储机制**: 与海马的情景记忆不同，纹状体不使用 ClusterNetwork。程序性记忆使用 S-R（刺激-反应）联结表——一个稀疏的 `dict[(state_hash, action)] → strength` 映射。这对应基底节的解剖特征：纹状体不是细胞集群编码，而是中型多棘神经元的突触权重矩阵。

- **D1 直接通路 (Go)**: 感知状态 → 促进特定动作，多巴胺↑ → 增强
- **D2 间接通路 (No-Go)**: 感知状态 → 抑制特定动作，多巴胺↓ → 增强
- **习惯强度**: [0,1] 渐进自动化。自动化程度 = 该状态-动作对的重复次数 / 总尝试次数
- **去价值化诊断**: 非运行时功能，仅用于测试/验证——模拟结果贬值后，习惯行为是否仍持续（这种行为僵化是习惯的定义特征）
- **海马-纹状体竞争**: 高新颖性→海马主导（灵活推理），低新颖性→纹状体主导（自动化高效）

### 整合

```
VTA ──da_signal──► Striatum ──habit_action──► MoEGate
                     ▲                          │
前额叶 ──goal_action─┘                   最终动作选择 (竞争)
```

## 第四节：干扰遗忘

在已有被动衰减的基础上，加入三种主动遗忘：

| 机制 | 实现 | 对应综述 |
|------|------|---------|
| 前摄干扰 | learn() 时检测桶内相似簇数量 → 相似簇多则降低学习率 | §1.4.1(1) |
| 倒摄干扰 | learn() 后对桶内高相似旧簇施加轻微衰减 | §1.4.1(2) |
| 提取失败 | recall() 加入情境匹配权重（编码特异性原则）| §1.4.1(3) |

不增加新存储结构，仅修改 `ClusterNetwork.learn()` 和 `recall()` 的行为参数。

### 启动学习过程

```
第 1 轮: 用户输入 → 0 簇 → 创建第一个簇 → 无回应能力
第 10 轮: 同一输入 → recall 命中 → 模仿/回声
第 100 轮: 直接命中 → 稳定回应模式 → 形成习惯
```

### 启动配置

```python
# v6.0 纯净模式 Theta 调整（适配零预训练）
PURE_MODE_THETA = {
    'cluster_threshold': 0.65,   # 降低匹配阈值（初始簇少, 需放宽）
    'learn_rate_l0': 0.12,       # 提高初始学习率（快速吸收新知识）
    'decay_rate': 0.005,         # 降低衰减（避免早期记忆过快消失）
    'exploration_bonus': 0.3,    # 提高探索（鼓励主动学习）
    'critical_window': 5000,     # 延长关键期（语言学习敏感期）
}
```

## 与已有模块的关系

所有新增模块**不替换**现有模块，而是**叠加新的记忆子系统**：

```
v5.7 数据流（保持不变）:
  s → visual/auditory/nociception hierarchy → hypothalamus → VTA → LC
    → L0.learn(s) → ACC.compute_F → TPN seesaw → FPN gate → PFC action

v6.0 新增记忆旁路（并行叠加）:
  每次 learn(s) 同时:
    → agent.net (情景, 海马)         — 已有
    → semantic.learn(s) (语义, 皮层)  — 新增 (慢速并行学习)
    → striatum.learn(s,a,r) (习惯)    — 新增 (试错反馈学习)
  
  每次 recall(s) 同时:
    → 海马情景检索 (已有)
    → 语义知识检索 (新增)
    → 纹状体习惯检索 (新增)
  
  会话结束时:
    → 系统巩固: 海马→语义提取 (新增)
    → 跨会话持久化 (增强)
```

## 验收标准

1. **零预训练启动**: 删除 corpus.txt 后 Agent 能正常启动，0 簇初始状态
2. **在线学习**: Agent 能在对话中逐词学习，词汇量随对话轮次增长
3. **跨会话记忆**: 关闭重启后，之前学到的知识保留（语义记忆持久化）
4. **习惯形成**: 重复相同的对话模式后，Agent 回应速度提高（纹状体习惯）
5. **主动遗忘**: 相似信息过多时出现干扰效应，情境不匹配时提取困难
6. **系统巩固**: 会话结束后，情景记忆被提取为语义知识
7. **程序性学习**: Agent 的对话技能随练习改善（速度/流畅度）
8. **不违背九条原则**: 所有机制均基于 Hebb 学习 + 自由能最小化
