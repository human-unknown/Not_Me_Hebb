Module B: 亮度/颜色恒常性 — 三种实现方式

目标: 同一物体在不同光照下产生一致的 Gabor 编码，减少集群碎片化

A

前置归一化 (Pre-Gabor Normalization)

在 Gabor 编码之前对原始图像做归一化：DoG 局部对比度 + Gray World 白平衡。类似相机 ISP 管线。

Pros

  • 改动最小 — 只加一个 _preprocess_constancy() 方法
  • 所有后续编码自动受益
  • 计算快 (O(N) 线性)

Cons

  • 不太"生物" — 视网膜/LGN 做的是局部增益控制，不是全局白平衡
  • Gray World 假设在某些场景失效（如纯色物体主导）
B

后置归一化 (Post-Gabor Gain Control) ★ 推荐

在 Gabor 响应之后，对每个滤波器的响应做局部对比度归一化（divisive normalization）。模拟 V1 的 surround suppression / gain control。

Pros

  • 生物精确 — V1 简单细胞确实有 surround suppression
  • 与现有 Hebb gain 可以统一 (gain = 可塑性 + 归一化)
  • 对局部和全局光照变化都鲁棒
  • 符合 FEP — 归一化 = 降低冗余预测误差

Cons

  • 需要修改 encode() 内部逻辑，比 A 复杂
  • 需要调 surround 半径参数
C

双通路 (Dual-Path: Pre + Post)

同时做前置全局白平衡 + 后置局部 divisive normalization。最完整但最重。

Pros

  • 最鲁棒 — 覆盖全局和局部两种光照变化
  • 类似生物双通路：Parvo(颜色) + Magno(亮度)

Cons

  • 计算量最大
  • 可能与现有 Hebb gain 产生交互副作用
  • 过度归一化可能抹掉有用的亮度信息

B 方案细节 (推荐): 生物启发的 Divsive Normalization

数学: R_norm[i,x,y] = R[i,x,y] / (σ² + Σ_j w(j) * R[j,x,y]²)
R[i,x,y] = |Gabor 响应| (滤波器 i, 位置 x,y)
R_norm = R[i,x,y] / (σ² + Σ_{j∈neighborhood(i)} Σ_{dx,dy∈surround} w(j,dx,dy) × R[j,x+dx,y+dy]²)

其中:
σ² = 半饱和常数 (默认 0.1)
neighborhood(i) = 同尺度的 4 个相邻朝向滤波器
surround = 3×3 空间邻域
w = 高斯权重 (σ_surround = 1.5 像素)

对颜色通道的处理

在 encode_color() 中额外加一步：对 RG/BY opponent 响应做相同的 divisive normalization，实现颜色恒常性——这是现有管线完全没有的。
